# Dataset Discovery

Generated by `python -m tracegate data discover`.

## Selection Summary

Final selected source: GitHub REST API Pull Requests. The P0 path uses public Pull Request metadata and changed-file provenance directly from GitHub, with no mock, synthetic, or fallback records.

## Candidates

### GitHub REST API Pull Requests

- Official URL / dataset URL: https://docs.github.com/en/rest/pulls/pulls
- Data fields: PR title/body/state/merged_at/base/head/merge_commit_sha plus changed files from the PR files endpoint.
- License / use restrictions: Public GitHub repository content remains under each repository's license; API use follows GitHub terms.
- GitHub token required: Optional for public data; unauthenticated rate limit is lower.
- Extra dependencies: Python standard library urllib/json only.
- Download method: GET /repos/{owner}/{repo}/pulls and GET /repos/{owner}/{repo}/pulls/{pull_number}/files.
- Sample scale: P0 target 12 records, minimum 8 scored cases.
- P0 decision: selected

### SWE-bench / SWE-bench Lite / Verified

- Official URL / dataset URL: https://www.swebench.com/
- Data fields: Issue text, repo, base commit, patch/test patch metadata depending on split.
- License / use restrictions: Dataset-specific terms; source repos keep their own licenses.
- GitHub token required: No GitHub token for published metadata, but dataset tooling can add dependencies.
- Extra dependencies: Often HuggingFace datasets/pandas/pyarrow for convenient access.
- Download method: Not selected for P0 to avoid heavyweight dependencies and large downloads.
- Sample scale: Large benchmark; P0 would need curated small subset.
- P0 decision: candidate_not_selected

### Agent-authored / AIDev PR datasets

- Official URL / dataset URL: Public dataset discovery required per source.
- Data fields: Potential PR metadata and agent traces if available.
- License / use restrictions: Varies by dataset.
- GitHub token required: Varies.
- Extra dependencies: Unknown until source selection.
- Download method: Not selected for P0 because availability and provenance vary.
- Sample scale: Unknown.
- P0 decision: candidate_not_selected

## Sampling Strategy

- Fetch recent closed Pull Requests from a small fixed list of public repositories.
- Keep only merged Pull Requests with changed-file provenance.
- Normalize each merged PR into a claim/evidence case with `source_dataset=github_api`.
- Mark low-provenance records as `needs_manual_review` and exclude them from scored metrics.

## Cache Policy

- Raw API responses are written to `datasets/real_min/raw/`, which is ignored by Git.
- Normalized real cases may be committed only when they are small, public, and provenance-linked.
- The manifest records sha256 for the normalized dataset.
- If fetch fails or produces fewer than 8 scored real cases, no fallback data is generated.

## Cache Verification

A cache is considered real only when it contains public GitHub URLs, merge commit or PR provenance, at least one evidence item per scored case, and a manifest hash matching the dataset used by the run.
